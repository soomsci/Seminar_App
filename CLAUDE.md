# 공통 지침서 — 전 세션 필독

이 프로젝트는 여러 Claude Code 세션이 역할을 나눠 병렬로 개발한다.
**작업 시작 전에 반드시 `docs/SPEC.md`(명세서)와 `docs/TASKS.md`(내 역할)를 읽어라.**

## 프로젝트 한 줄 요약

교사 연수 실습에서 연수생이 🟢완료/🔴막힘 버튼을 누르면, 강사 대시보드에 색깔 원 격자와 진행률(%)로 실시간 수합되는 웹앱. 세션코드로 입장.

## 절대 규칙

1. **자기 소유 파일만 수정한다.** 파일별 소유권은 SPEC.md §6에 있다. 다른 세션 소유 파일을 고쳐야 한다면 수정하지 말고 `docs/STATUS.md`에 요청 사항을 남겨라.
2. **명세를 임의로 바꾸지 않는다.** SPEC.md와 다르게 만들고 싶으면 먼저 STATUS.md에 제안을 남기고 총괄매니저(사용자) 승인을 받는다. 특히 `db.js`의 API 시그니처(SPEC §5)는 계약이다.
3. **기능을 늘리지 않는다.** SPEC §8(Out of Scope)에 있는 것은 "있으면 좋을 것 같아도" 만들지 않는다. 이 앱의 핵심 가치는 단순함이다.
4. **작업을 마치면 `docs/STATUS.md`의 자기 섹션을 갱신한다.** 형식: 완료한 것 / 남은 것 / 다른 세션에 요청·공유할 것. 다른 세션과의 소통은 이 파일로만 한다.

## 기술 규칙

- 순수 HTML/CSS/JS (ES Modules). **빌드 도구, 프레임워크, npm 패키지 금지.** 외부 의존은 Firebase CDN SDK만.
- Firebase 접근은 `js/db.js`를 통해서만. B·C 세션은 firebase SDK를 직접 import하지 않는다.
- `db.js`가 준비 안 됐거나 Firebase 키가 없어도 개발할 수 있도록 URL에 `?mock=1`을 붙이면 목 모드로 동작한다(SPEC §5). UI 세션은 목 모드로 개발·확인하라.
- 문자 인코딩 UTF-8, 화면의 모든 문구는 한국어.
- 코드 주석은 꼭 필요한 제약 설명에만, 한국어로 간결하게.
- 스타일: `css/common.css`의 CSS 변수(색상 토큰)를 사용한다. 상태 색은 `--c-green`, `--c-red`, `--c-gray`로 통일.
- 로컬 확인 방법: `python3 -m http.server 8000` 후 `http://localhost:8000/public/...?mock=1`

## 품질 기준

- 연수생 화면: 320px 모바일에서 깨지지 않을 것, 버튼은 한 손 엄지 조작 기준.
- 강사 대시보드: 프로젝터에 띄웠을 때 뒷자리에서 원 색이 구분될 것.
- 에러는 사용자에게 한국어로 안내한다(예: "세션코드를 찾을 수 없어요"). console.log를 사용자 안내 대용으로 쓰지 않는다.
- 완료 보고 전에 반드시 브라우저(목 모드)로 직접 열어 동작을 확인한다.
